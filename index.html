      • 去均值 Mean subtraction
        • X -= np.mean(X, axis = 0)
      • 归一化 Normalization
        • X /= np.std(X, axis = 0)
      • PCA /白化 Whitening :首先均值化数据，然后求协方差矩阵
        • X -= np.mean(X, axis = 0) //求均值
        • cov = np.dot( X.T, X) / X.shape[0] // 求协方差矩阵
        • U,S,V = np.linalg.svd(cov) //奇异值分解,其中 U 为特征向量，S为奇异值分解向量为特征向量的平方
        • Xrot = np.dot(X, U) //去除数据相关性
        • Xrot_reduced = np.dot(X, U[:,:100]) 降维到 100 维
        • Xwhite = Xrot / np. sqrt(s + 1e-5) //白化数据 1e-5 为防止除0
    • 正则化L1L2区别
      • L1：生成稀疏权值矩阵，特征提取
      • L2：降低过拟合

  • 网站存在sql注入点，判断数据库类型

    • SQL的注入类型

      • Boolean-based blind SQL injection（布尔型注入）
      • Error-based SQL injection（报错型注入）
      • UNION query SQL injection（可联合查询注入）
      • Stacked queries SQL injection（可多语句查询注入）
      • Time-based blind SQL injection（基于时间延迟注入）

    • sql注入判断数据库类型方法-回显注入

      • 基于特定函数

        • and len(‘a’)=1
          • 返回正常页面时，可以推断当前的数据库类型可能是mssql，或mysql，或是db2
        • and lenth(‘a’)=1
          • oracle和informix
        • version()>1 返回与@@version>1 相同页面时，则可能是mysql。如果出现提示version()错误时，则可能是mssql。
        • 在mssql中可以调用substring。oracle则只可调用substr

      • 基于辅助符号

          • 以上参数都是int的时候，如果是字符型的话首先在参数后面加上单引号
        • and exists (select count(*) from sysobjects)
and exists (select count(*) from msysobjects)

          如果第一条返回正常，就是MSSQL数据库，如果两条都不正常，那就是ACCESS数据库了。

          第一句意思是查询sysobjects表里记录数大于,返回正常的，说明大于0且存在sysobjects这个表，因为这个表只有MSSQL数据库才有，所以可以判断为MSSQL数据库。返回错误则表示不是。

          第二句提交是不会返回正常页面的，就算是ACCESS数据库也不会返回正常。因为默认情况下我们没有权限查询这个表里的数据。WEB会提示我们“记录无法读取;’msysobjects’没有读取权限”，如果返回的是这个错误信息的话，那就证明是ACCESS数据库了

      • 注入点后直接加上单引号，根据服务器的报错信息来判断数据库

        • Microsoft JET Database Engine 错误 ‘80040e14’，说明是通过JET引擎连接数据库->ACCESS
        • ODBC->MSSQL

    • sqlmap 如何绕过WAF

      • tamper，需要确定系统过滤了哪些关键字，url参数中手工带入关键词，判断是否被过滤。
        • 空格：space2comment.py
        • 大小写敏感： randomcase.py
        • ’>‘：greatest.py
        • ‘=’: equaltolike.py
        • 双引号: apostrophenullencode.py
        • ‘#’: space2hash.py
      • url编码，如蚁剑的编码器

    • 如何得到WAF的web日志中某种攻击的误报率（数据挖掘）

      • WAF度量标准
        • 真实的攻击被阻断(TP)
        • 有效的请求允许通过(TN)
        • 有效的流量被不恰当的阻断(FP)
        • 攻击被允许通过(FN)
      • 模糊匹配（正则）

    • 网络的五元组：源IP地址、目的IP地址、协议号、源端口、目的端口

      • 四元组：源IP地址、目的IP地址、源端口、目的端口
      • 七元组：源IP地址、目的IP地址、协议号、源端口、目的端口，服务类型以及接口索引

    • 流量分析

      • webshell入侵检测

密码学原理

0x01

  • 五元组（P,C,K,E,D）

    • Plaintext
    • Ciphertext

  • 攻击模型

    • 唯密文攻击
    • 已知明文攻击
    • 选择明文攻击
    • 选择密文攻击（强度最大）

  • 密码分析：Kerckhoff假设

  • 古典密码（单射）

    • 代换（改变字符）
      • 实例
        • 棋盘密码
        • 凯撒密码
        • 曾公密码
      • 类别
        • 仿射密码
          • 单表代换
          • a=1，移位密码
            • 若密钥以相同概率随机使用，则具有完善保密性
          • a=1，b=3，凯撒密码
          • 不能抵抗唯密文攻击，根据密文得到置换
        • 希尔HILL密码
          • 多字母代换
          • 不能抵抗已知明文攻击，根据明文获得矩阵长度
        • 维吉尼亚密码
          • 多表代换
          • 不能抵抗唯密文攻击，根据密文得到密钥长度
            • kasiski法
            • 重合指数法（0.065）
    • 置换（改变位置）
      • 希尔密码是特殊形式的置换密码
    • 流密码
      • 维吉尼亚密码是短周期的同步流密码
      • 周期流密码
      • 非周期流密码
        • 弗纳姆Verman密码
          • 一次一密
      • 可用线性递推关系产生伪随机序列
        • LFSR线性反馈移位寄存器
        • 不能抵抗已知明文攻击，根据明文获得LFSR的结构
      • 同步流密码
        • 存在周期问题
      • 异步流密码
      • 自动密钥密码
    • tips
      • e频率最高
      • gcd最大公约数
      • -
      • 可用分组密码生成密钥
    • 本原多项式

0x02

  • 密码体系安全性的评判

    • 计算安全性

    • 可证明

      • 归约

    • 无条件安全

      • 提供无限计算资源也无法被攻破
      • 完善保密性
        • 不能通过密文得到明文
        • 先验概率= 后验概率
          • 先验概率，如全概率；后验概率，如贝叶斯公式
        • |K|>=|C|>= |P|，单射
        • 证明一次一密不可破解
          • 假设|K|=|C|=|P|，该密码体制是完善保密的，当且仅当密钥使用概率都是1/|K|,且PC存在唯一密钥，使得ek(x)=y

    • dk1(y)=x 且 dk2(y)=x 不能确定k1 = k2

      • 仿射
      • playfair密码
